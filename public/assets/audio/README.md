# SCUTTLE audio — manifest + drop-in convention

Most sounds here are now **real recordings** (trimmed/split/converted via ffmpeg). A few
minor cues are still **procedural placeholders** generated by `scripts/gen-audio.mjs`
(`node scripts/gen-audio.mjs` regenerates ONLY those — it no longer recreates the real ones).
Provenance + licenses: `../CREDITS.md`.

## How the engine finds sounds

`src/render/audio.ts` holds a `MANIFEST` mapping each sound **id** to a path — **or an
array of paths** (interchangeable takes; the engine plays a random one each time, never the
same twice running). The id is what gameplay code plays (e.g. `audio.playAt("cannon", pos)`).
To upgrade a sound, do **either**:

1. **Same name:** drop your replacement in at the exact path below (keep the filename), or
2. **New file / add a variant:** add the file and change/extend that id's entry in the `MANIFEST`.

Positional SFX (cannon, impact, crunch, creak, rope, sink) **must be mono** or the 3D panner
won't spatialize them. Ambience and music may be stereo. `.ogg` is preferred for size; the
loader decodes whatever the browser supports (`.ogg`, `.wav`, `.mp3`). A path used by several
ids decodes once (the loader dedupes).

## Manifest (id → path)

| id            | path(s)                                  | kind / notes |
|---------------|------------------------------------------|--------------|
| cannon        | `sfx/cannon_1..4.ogg`                     | positional — **4 random takes** |
| impact_wood   | `sfx/wood_crack_1..4.ogg`                 | positional — shared damage pool |
| impact_thud   | `sfx/wood_crack_1..4.ogg`                 | positional — shared damage pool |
| crunch        | `sfx/wood_crack_1..4.ogg`                 | positional — shared damage pool |
| sink          | `sfx/sink.wav`                            | positional/2D — *placeholder* |
| coins         | `sfx/coins.ogg`                           | 2D (UI)      |
| splash        | `sfx/splash.wav`                          | positional — *placeholder*, unused |
| gull          | `sfx/gull.wav`                            | positional — *placeholder*, **MUTED** |
| creak         | `sfx/creak.ogg`                           | positional (roll/heel + rudder) |
| rope          | `sfx/rope_1..3.ogg`                        | positional (sail trim) — **3 random takes** |
| ui_click      | `sfx/ui_click.mp3`                        | 2D (UI)      |
| ui_confirm    | `sfx/ui_confirm.wav`                      | 2D (UI) — *placeholder* |
| ui_buy        | `sfx/ui_buy.mp3`                          | 2D (UI)      |
| port_open     | `sfx/port_open.ogg`                       | 2D (UI) — ship's bell |
| ship_ready    | `sfx/ship_ready.wav`                      | 2D (UI) — *placeholder* |
| ocean_loop    | `ambient/ocean_loop.ogg`                  | looping bed  |
| wind_loop     | `ambient/wind_loop.ogg`                   | looping bed (wind + creaking) |
| menu_theme    | `music/menu_theme.ogg`                    | music — menu + pause |
| sea_ambient   | `music/sea_ambient.wav`                   | *placeholder*, **unused** (see below) |
| harbor        | `music/harbor.ogg`                        | music — at port |

> **Music plays on the menu/pause screen and at port. At sea there is NO music** — open-water
> sailing is ambience-only (wind + ocean), which is why `sea_ambient` is unused and
> `musicTrackForState("playing")` returns `""`. To score the sailing phase, drop a track in
> and point `sea_ambient` at it (or return its id from `musicTrackForState`).
>
> **`gull` is MUTED** (the synthetic seagull was unpleasant) — listed in `PLACEHOLDER_MUTED`
> in `audio.ts`. Drop a real gull recording at `sfx/gull.wav`, delete `gull` from that set, and
> it plays again — the in-game trigger is already wired.

  but list them in `../CREDITS.md` anyway.
